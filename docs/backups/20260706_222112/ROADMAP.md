# Roadmap

这个 Roadmap 主动暴露不足，但每个不足都有明确收敛方向，避免面试时显得“只能 Demo，不能推进”。

## P0：让演示更稳

- 补齐截图：知识库列表、上传 metadata、检索 trace、聊天引用、拒答、评测页。
- 固化一套可复现实验数据：演示账号、知识库、源 DOCX、评测结果、截图命名保持一致。
- 给评测结果导出 JSON，避免前端截图成为唯一证据。
- 增加一键 smoke test：登录、知识库列表、评测 latest、检索接口、trace 字段完整性。

## P1：提升检索质量

- 扩展评测集到 100-200 条，按后勤、医工、跨文档、负例分层。
- 为 parent_id 标注增加校验脚本，文档重建后自动发现失效标注。
- 对 parent_size、child_size、child_overlap、RRF rank_constant、rerank TopN 做小规模网格实验。
- 为 Router 增加错误样本回放，统计错路由、漏路由、过宽路由。
- 将拒答阈值按问题类型和部门风险分级，而不是全局单阈值。

## P2：生产化能力

- 文档解析接入 OCR 和版面分析，支持扫描件、图片表格、复杂附件包。
- 建立增量索引任务队列，替换当前 background task，支持失败重试、取消和并发限流。
- 将 RAG trace 接入审计后台，支持按用户、知识库、部门、拒答原因检索。
- 增加人工反馈闭环：引用是否正确、是否误拒答、是否需要补文档。
- 引入权限策略中心，把部门、角色、数据范围、文档密级统一建模。

## P3：医疗场景扩展

- 增加医疗设备型号、院区、科室、合同/维保周期等结构化字段。
- 对敏感问题建立安全分类：隐私、凭据、合同财务、未授权医疗建议、院内流程。
- 对接院内 SSO、审计日志、堡垒机、私有模型网关。
- 对答案增加“适用范围”和“资料版本”提示，避免旧制度被误用。

## 已知不足

| 不足 | 风险 | 收敛方式 |
| --- | --- | --- |
| 评测集仍偏 Demo 规模 | 分数不能代表生产全量 | 分层扩充评测集，保留每类指标 |
| 拒答规则包含显式触发词 | 对新型负例泛化有限 | 引入安全分类器和人工反馈调参 |
| Milvus sparse 依赖版本能力 | 部署环境不一致时可能降级 | 保留 SQL lexical fallback，部署文档固定版本 |
| 文档解析主要覆盖 DOCX/PDF 文本 | 扫描件和复杂表格不足 | 接 OCR/版面解析与表格结构化 |
| Router 默认可关闭且依赖 profile 质量 | profile 差会影响候选库 | 入库后自动刷新 profile，增加路由评测 |

## 下一步最值得做的事

先补齐 P0 截图和导出报告，再扩展评测集。这样面试时既能讲工程链路，也能拿出“我知道不足、且能验证优化是否有效”的证据。
