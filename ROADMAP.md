# Roadmap

## P0：稳定可复现

- 补齐上传 metadata 预览截图，解决前端自动上传状态与接口返回不一致问题。
- 导出评测报告 JSON，避免只依赖页面截图。
- 增加 smoke test：登录、知识库列表、检索接口、评测 latest、trace 字段完整性。
- 固化一套可复现数据：知识库、源文档、评测集、截图命名保持一致。

## P1：提升检索质量

- 将评测集扩展到 100-200 条，按后勤、医工、跨文档、负例分层统计。
- 增加 parent_id 标注校验脚本，文档重建后自动发现失效标注。
- 对 parent_size、child_size、child_overlap、RRF rank_constant、rerank TopN 做参数实验。
- 建立 Router 错误样本回放，区分错路由、漏路由和过宽路由。
- 将拒答阈值按问题类型和风险等级配置。

## P2：生产化能力

- 文档解析接入 OCR 和版面分析，支持扫描件、图片表格、复杂附件包。
- 将 background task 升级为任务队列，支持失败重试、取消、并发限流和进度追踪。
- RAG trace 接入审计后台，支持按用户、知识库、部门、拒答原因检索。
- 增加人工反馈闭环：引用是否正确、是否误拒答、是否需要补文档。
- 引入权限策略中心，统一建模部门、角色、数据范围和文档密级。

## P3：医疗场景扩展

- 增加设备型号、序列号、维保周期、合同、配件、故障码等结构化字段。
- 支持院区、楼宇、科室、点位维度过滤。
- 对制度和手册增加版本号、有效期和过期提示。
- 接入院内 SSO、私有模型网关、审计日志和统一告警。

## 已知不足

| 不足 | 风险 | 收敛方式 |
| --- | --- | --- |
| 评测集规模仍有限 | 分数不能代表生产全量 | 分层扩充评测集 |
| 拒答包含显式触发规则 | 新型负例泛化有限 | 引入安全分类器和人工反馈 |
| Milvus sparse 依赖版本能力 | 环境差异可能导致降级 | 固定部署版本并保留 fallback |
| 文档解析主要覆盖文本型 DOCX/PDF | 扫描件和复杂表格不足 | 接 OCR、版面解析和表格结构化 |
| Router 依赖 KB profile 质量 | profile 差会影响候选库选择 | 入库后自动刷新并增加路由评测 |
